# 手動テストプラン — ASIO 入力サポート

本ドキュメントは、実機 ASIO ハードウェアが利用可能な環境で実施する手動検証手順を記載する。

## 前提条件

- **Unity バージョン**: Unity 6000.3.10f1
- **スクリプティングバックエンド**: Mono
- **OS**: Windows 10/11 x86_64
- **ASIO ドライバ**: 1 件以上インストール済み（例: ASIO4ALL、Steinberg UR シリーズ、RME 等）
- **ASIO 対応オーディオインターフェース**: マイク入力が可能なデバイスが接続されていること
- **uLipSync プロファイル**: キャリブレーション済みの `uLipSyncProfile` が用意されていること

---

## テストシナリオ

### 1. ASIO デバイス列挙

| # | 手順 | 期待結果 |
|---|------|----------|
| 1.1 | Unity Editor で `uLipSyncAsioInput` コンポーネントをゲームオブジェクトに追加する | インスペクタにコンポーネントが表示される |
| 1.2 | インスペクタの ASIO デバイスドロップダウンを確認する | インストール済みの ASIO ドライバ名が一覧に表示される |
| 1.3 | 「デバイス更新」ボタンを押す | デバイスリストが再取得され、最新のドライバ一覧が反映される |
| 1.4 | ASIO ドライバが 1 件もインストールされていない環境で同様に確認する（→ シナリオ 5.1 参照） | 空リストと「ASIO ドライバが見つかりません」警告が表示される |

**対応要件**: 1.1, 1.2, 1.3, 1.4, 1.5

---

### 2. 録音開始・停止

| # | 手順 | 期待結果 |
|---|------|----------|
| 2.1 | `isAutoStart` を `false` に設定し、Unity Editor で再生を開始する | 録音は自動開始されない。`isRecording` が `false` のまま |
| 2.2 | インスペクタの「Start Recording」ボタンを押す | 録音が開始され、`isRecording` が `true` になる |
| 2.3 | インスペクタの「Stop Recording」ボタンを押す | 録音が停止し、`isRecording` が `false` になる |
| 2.4 | 停止後に再度「Start Recording」を押す | 新しい `AsioOut` インスタンスで録音が正常に再開される |
| 2.5 | `isAutoStart` を `true` に設定し、Unity Editor で再生を開始する | 録音が自動的に開始され、`isRecording` が `true` になる |
| 2.6 | 録音中に再生を停止する（`OnDisable` / `OnDestroy` 経路） | 録音が自動停止し、リソースリークが発生しない（Console にエラーなし） |

**対応要件**: 2.1, 2.2, 2.3, 2.4, 2.5, 2.6, 2.7

### 3. リップシンク動作確認（フォネームイベント発火）

| # | 手順 | 期待結果 |
|---|------|----------|
| 3.1 | `uLipSyncAsioInput` の `lipSync` フィールドに `uLipSync` コンポーネントを割り当て、キャリブレーション済みプロファイルを設定する | インスペクタで正しくリンクされている |
| 3.2 | 録音を開始し、マイクに向かって発声する（あ、い、う、え、お） | `uLipSync` のフォネーム結果がリアルタイムに更新される |
| 3.3 | `uLipSyncBlendShape` 等の出力コンポーネントを接続して確認する | アバターの口がリアルタイムに動く |
| 3.4 | 発声を止めた状態で確認する | フォネーム結果がニュートラルに戻る |
| 3.5 | 既存の `uLipSyncMicrophone` 経由の動作と比較する | ASIO パスでもフォネーム解析結果が同等に動作する |

**対応要件**: 4.1, 4.2, 4.5, 4.6

### 4. サンプルレート補正確認

| # | 手順 | 期待結果 |
|---|------|----------|
| 4.1 | ASIO デバイスのサンプルレートを 44100 Hz に設定して録音・リップシンクを確認する | フォネーム解析が正常に動作する |
| 4.2 | ASIO デバイスのサンプルレートを 48000 Hz に設定して録音・リップシンクを確認する | フォネーム解析が正常に動作する |
| 4.3 | ASIO デバイスのサンプルレートを 96000 Hz に設定して録音・リップシンクを確認する | フォネーム解析が正常に動作する（高サンプルレート補正が機能している） |
| 4.4 | Unity の `AudioSettings.outputSampleRate` と ASIO デバイスのサンプルレートが異なる組み合わせで確認する | ASIO 由来のサンプルレートが `ScheduleJob` に正しく伝播し、解析結果に不整合がない |

**対応要件**: 3.4, 3.5, 7.5, 7.6

### 5. エラーケース

#### 5.1 ドライバ不在

| # | 手順 | 期待結果 |
|---|------|----------|
| 5.1.1 | ASIO ドライバが未インストールの Windows 環境で `uLipSyncAsioInput` を追加する | デバイスリストが空。「ASIO ドライバが見つかりません」警告が表示される |
| 5.1.2 | ドライバ不在の状態で再生を開始する（`isAutoStart = true`） | `Debug.LogError` でエラーメッセージが出力され、`isRecording` は `false`。クラッシュしない |

**対応要件**: 1.3, 5.1

#### 5.2 デバイスビジー

| # | 手順 | 期待結果 |
|---|------|----------|
| 5.2.1 | 別のアプリケーション（DAW 等）で同じ ASIO デバイスを占有した状態で録音を開始する | 「デバイスが使用中です。他のアプリを終了してから再試行してください」趣旨のエラーメッセージが `Debug.LogError` で出力される。`isRecording` は `false` |
| 5.2.2 | エラー後、他のアプリを終了してから再度録音を開始する | 正常に録音が開始される |

**対応要件**: 5.2

#### 5.3 コールバック例外

| # | 手順 | 期待結果 |
|---|------|----------|
| 5.3.1 | 録音中に ASIO デバイスを物理的に切断する（USB オーディオインターフェースの場合） | `Interlocked` エラーフラグが設定され、次の `Update()` でエラーログが出力される。録音が安全に停止する。Unity がクラッシュしない |

**対応要件**: 4.3, 4.4, 5.3, 5.4

### 6. チャンネル設定

| # | 手順 | 期待結果 |
|---|------|----------|
| 6.1 | `inputChannelOffset = 0`, `inputChannelCount = 1` で録音・リップシンクを確認する | 最初のチャンネルからサンプルが取得され、リップシンクが動作する |
| 6.2 | マルチチャンネル ASIO デバイスで `inputChannelOffset` を変更し、別の入力チャンネルを選択する | 指定したチャンネルからサンプルが取得される |
| 6.3 | `inputChannelOffset + inputChannelCount` がデバイスの最大チャンネル数を超える値を設定する | `Debug.LogWarning` が出力され、利用可能範囲にクランプされる。クラッシュしない |

**対応要件**: 3.1, 3.3, 3.6

## 合否判定基準

- 全シナリオで Unity がクラッシュしないこと
- 全シナリオで期待結果と一致すること
- Console に予期しないエラーメッセージが出力されないこと
- 既存の `uLipSyncMicrophone` 経由のリップシンク動作に影響がないこと
